# Outreach Debate Launch Checklist

**Last updated:** 2026-05-11  
**Owner:** Edward  
**Purpose:** Step-by-step execution guide for turning the 30 prebuilt outreach debates into a live cold outreach campaign.  
**Reference docs:**  
- Debates: `docs/outreach-debates/<name>.md` (30 files)  
- Email templates: `docs/phase0-cold-email-templates.md`  
- Target list: `docs/phase0-target-list.md`  
- Pricing: `docs/pricing.md`

---

## Overview

30 debate files are ready. The campaign sends personalized cold emails that surface each person's prebuilt debate as a gift. The send sequence is 3 emails per recipient (initial + 2 follow-ups) across 9 days. Total pipeline: 30 people × 3 touches.

## Phase 1 — Pre-Launch Validation

Complete every item before sending a single email.

### 1.1 Debate File Audit (per file)

Run through each of the 30 debate files and verify all of the following:

- [ ] File opens with the correct frontmatter block:
  - `# Agora Debate: <Full Name>`
  - `**For:** <Name> (<description>)`
  - `**Topic:** <their specific decision>`
  - `**Council:** <3 named minds>`
  - `**Generated:** <date>`
- [ ] Contains Rounds 1, 2, and 3 with responses from all 3 council members
- [ ] Contains a `## Council Consensus` section with all 4 sub-sections:
  - `### Consensus Points`
  - `### Key Tensions`
  - `### Recommended Action`
  - `### Immediate Next Steps` (3 numbered steps)
  - `### Risk Factors`
- [ ] `### Recommended Action` paragraph is tight enough to use as the inline excerpt in Variant A emails (2-3 sentences max; if it runs longer, copy the sharpest 2-3 sentences)
- [ ] Footer line: `*Generated by [Consult The Dead — The Agora](https://consultthedead.com/agora)*`
- [ ] No placeholder text remaining (`{{...}}`, `TBD`, `[INSERT]`)

**Known gap to check:** Some debate files may lack the `### Risk Factors` sub-section or have a `### Immediate Next Steps` section with fewer than 3 steps. Note any gaps below but do not block the send — gaps in debate files do not affect email deliverability.

- [ ] No two recipients have identical subject lines

## Phase 3 — Send Sequence

### 3.1 Day 1 — Initial Send (Batch 1: 20-30 emails)

Send order recommendation: prioritize by recency of their decision signal. Founders currently wrestling with a live decision convert better than those who decided 6+ months ago.

- [ ] Load the first batch (up to 30) into your sending tool, scheduled for a Tuesday–Thursday morning send (avoid Monday and Friday)
- [ ] Send between 8am–11am in the recipient's local timezone (or use Eastern as the baseline)
- [ ] Confirm each email has the correct debate link (`consultthedead.com/debates/<slug>`)
- [ ] Log send date in the tracking table below

### 3.2 Day 1 — Monitor for Immediate Bounces

Within 2 hours of sending:

- [ ] Check for hard bounces — remove bounced addresses from all follow-up queues immediately
- [ ] Check for immediate out-of-office replies — note if they mention a return date (may want to delay follow-up)
- [ ] Check for any immediate replies — respond within 4 hours (see Phase 4: Reply Handling)

### 3.3 Day 4 — Follow-up 1

Subject: `re: {{original_subject}}`

Body:
> Hey {{first_name}}, bumping this — did the council's take land at all? If it missed the mark, that's useful feedback too.
> 
> Edward

- [ ] Send only to recipients who did not reply to the initial email
- [ ] Do not send to anyone who hard-bounced or unsubscribed
- [ ] Log follow-up 1 send date in tracking table

### 3.4 Day 9 — Follow-up 2 (Final Touch)

Subject: `re: {{original_subject}}`

Body:
> Last note — if the timing's off, no worries. The tool is at consultthedead.com/agora if you ever want to run a decision through it yourself. 3 agons/day, no signup.
> 
> Edward

- [ ] Send only to recipients who did not reply to either previous email
- [ ] This is the last touch — no further follow-up after this
- [ ] Log follow-up 2 send date in tracking table

## Phase 4 — Reply Handling

### 4.1 Positive Reply (engaged, asked a question, shared their debate reaction)

- [ ] Respond within 4 hours during business hours
- [ ] If they share a current decision they're wrestling with: run a live debate for free, then reply with the link — this is the conversion hook
- [ ] If they comment on the debate content: engage specifically with which advisor's argument they found most compelling
- [ ] If they ask about Pro pricing: share the current pricing (`docs/pricing.md`) — $30/mo or $300/yr

### 4.2 Conversion Reply (asks how to sign up or share with others)

- [ ] Direct them to `consultthedead.com/agora` — free tier needs no signup
- [ ] For Pro: `consultthedead.com/pricing`
- [ ] Mark as "converted" in tracking table immediately

### 4.3 Opt-out Reply ("unsubscribe", "remove me", "stop emailing me")

- [ ] Acknowledge and remove from all sequences immediately — do not send any further follow-ups
- [ ] Log as "opted out" in tracking table

### 4.4 Bounce or Invalid Address

- [ ] Hard bounce: mark as "bounced" in tracking table, suppress from all future outreach
- [ ] Soft bounce: retry once; if it bounces again treat as hard bounce

## Phase 6 — Post-Campaign Review

Run this after all Day 9 follow-ups have been sent and 3 additional days have passed (Day 12 minimum).

### 6.1 Metrics to Capture

Record the following:

| Metric | Target | Actual |
|--------|--------|--------|
| Emails sent (total) | 30 | |
| Hard bounces | <3 | |
| Open rate (if trackable) | >40% | |
| Reply rate (total) | >15% (4+ replies) | |
| Reply rate — Variant A | benchmark | |
| Reply rate — Variant B | benchmark | |
| Reply rate — Variant C | benchmark | |
| Live debates run on reply | — | |
| Converted to Pro | — | |
| Opted out | — | |

### 6.2 Qualitative Review

- [ ] Which debate files generated the most engagement? Note the topic pattern.
- [ ] Which council compositions (e.g., Marcus + Machiavelli + Sun Tzu) resonated most in replies?
- [ ] Which subject line approach drove the highest open rate?
- [ ] Which variant body produced the most substantive replies (vs. one-word "cool" responses)?
- [ ] Were there any misses where the debate clearly didn't match the person's actual situation?

### 6.3 Decision: Continue or Revise

Based on results:

- **If reply rate > 15%:** The debate format is working. Scale to the next cohort — expand the target list per `docs/roster-expansion.md`.
- **If reply rate is 5–15%:** Revisit variant assignment. Consider splitting the lowest-performing variant into two sub-variants before the next send.
- **If reply rate < 5%:** The problem is likely the cold channel or the subject line, not the debate content. Test a different acquisition channel (e.g., direct DM on Twitter/IH instead of email) before investing in more debate files.

## Appendix: Debate File Format Reference

All debate files in this directory follow this structure. Use this as a reference when writing new debates or auditing existing ones.

```
# Agora Debate: <Full Name>
**For:** <Name> (<context, e.g., product + MRR>)
**Topic:** <their specific decision question>
**Council:** <Mind 1>, <Mind 2>, <Mind 3>
**Generated:** <YYYY-MM-DD>

---

## Round 1
### <Mind 1>
[~300-word argument with a named construct and a closing "If my logic wins..." commitment]

### <Mind 2>
[same structure]

### <Mind 3>
[same structure]

## Round 2
### <Mind 1>
[responds to other two, refines position]

### <Mind 2>
[same]

### <Mind 3>
[same]

## Round 3
### <Mind 1>
[holds firm or concedes specific points, ends with "The single most important thing you should take from this agon:"]

### <Mind 2>
[same]

### <Mind 3>
[same]

---

## Council Consensus
### Consensus Points
### Key Tensions
### Recommended Action
### Immediate Next Steps
- Step 1 (Day 1-2) — ...
- Step 2 (Day 3) — ...
- Step 3 (Day 4-7) — ...
### Risk Factors

---

*Generated by [Consult The Dead — The Agora](https://consultthedead.com/agora)*
```
